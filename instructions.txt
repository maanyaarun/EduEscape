2. INSTALLATION STEPS
================================================================================

STEP 1: Install Python
-----------------------
If you don't have Python installed:

For Windows:
1. Go to https://www.python.org/downloads/
2. Download the latest Python 3 installer
3. Run the installer
4. ‚ö†Ô∏è IMPORTANT: Check "Add Python to PATH" during installation
5. Click "Install Now"

   (If you don't have Homebrew, install it from https://brew.sh)

------------------------------------
1. Create a folder called "eduescape" on your Desktop
2. Inside "eduescape", create two folders:
   - "backend"

3. Copy all the provided files into their respective folders

Your structure should look like:

------------------------------------
1. Open Terminal (Mac/Linux) or Command Prompt (Windows)

2. Navigate to the backend folder:
   cd Desktop/eduescape/backend

3. Install all required libraries:
   pip install -r requirements.txt

   This will install:
   - FastAPI (the web framework)
   - Uvicorn (the server)
   - PyPDF2 (for reading PDFs)

   ‚è±Ô∏è This may take 1-2 minutes.

4. Verify installation:

================================================================================
3. RUNNING THE BACKEND SERVER
================================================================================

STEP 1: Navigate to Backend Folder
-----------------------------------
cd Desktop/eduescape/backend

STEP 2: Start the Server
-------------------------
Run this command:

uvicorn main:app --reload

You should see output like:
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.

‚úÖ Your backend is now running!

What does this mean?
- "uvicorn" is the server program
- "main:app" tells it to run the "app" from "main.py"
- "--reload" means it will restart automatically if you change the code
- The server is running at http://127.0.0.1:8000 (also called localhost:8000)

‚ö†Ô∏è Keep this terminal window open! Don't close it.

STEP 3: Test the Backend
-------------------------
Open a new browser tab and go to:
http://localhost:8000

You should see:
{"message":"Welcome to EduEscape API! Upload a PDF to get started."}

‚úÖ If you see this, your backend is working!

To see all available API endpoints, go to:
http://localhost:8000/docs

This shows the interactive API documentation.

7. TROUBLESHOOTING
================================================================================

PROBLEM: "Command 'python' not found"
SOLUTION: 
- On some systems, use "python3" instead of "python"
- On Windows, you may need to use "py" instead
- Make sure Python is added to your PATH

PROBLEM: "Module not found" errors
SOLUTION:
- Make sure you ran "pip install -r requirements.txt"
- Try: pip3 install -r requirements.txt
- Or install individually: pip install fastapi uvicorn PyPDF2

PROBLEM: "Address already in use" error
SOLUTION:
- Another program is using port 8000
- Either stop that program, or use a different port:
  uvicorn main:app --reload --port 8001
- Then update the API_URL in script.js to http://localhost:8001

PROBLEM: "CORS error" in browser console
SOLUTION:
- Use Option 2 to serve the frontend (Python HTTP server)
- Or install a CORS browser extension

PROBLEM: PDF upload fails
SOLUTION:
- Make sure the PDF is readable (not password-protected)
- Check backend terminal for error messages
- Try a different PDF file
- The PDF must contain actual text (not just scanned images)

PROBLEM: Frontend can't connect to backend
SOLUTION:
- Make sure backend is running (check terminal)
- Make sure API_URL in script.js matches your backend URL
- Try accessing http://localhost:8000 directly in browser

PROBLEM: "No questions generated" or questions seem generic
SOLUTION:
- This is normal! Without AI, the questions are automatically generated
  based on simple text processing rules
- The questions will ask about the main topic and key terms found in the text
- For better questions, make sure your PDF has clear, well-structured text

PROBLEM: Answers marked wrong even when correct
SOLUTION:
- The answer checking is simple keyword matching
- Try to include key terms from the topic in your answer
- Look at the hints for guidance on what keywords to include

================================================================================
                     HOW THE QUESTION GENERATION WORKS
================================================================================

Since this version doesn't use AI, questions are generated using:

1. KEYWORD EXTRACTION
   - The system finds the most frequent important words in the text
   - Common words like "the", "and", "is" are filtered out
   - The top 5 keywords become the focus of questions

2. SUMMARY CREATION
   - Takes the first 3-4 sentences from the content
   - This gives students a quick overview to study

3. QUESTION TEMPLATES
   - Question 1: Asks about the main focus of the topic
   - Question 2: Asks about relationships between key concepts
   - Question 3: Asks about real-world applications

4. ANSWER CHECKING
   - Looks for key terms from the expected answer in student's response
   - If 30% or more of key terms match, answer is marked correct
   - This is flexible enough to accept various phrasings

5. HINTS
   - Hint 1: Shows the key terms to focus on
   - Hint 2: Reminds student to review the summary

This approach works well for basic active recall practice, though it's
simpler than AI-generated content.
